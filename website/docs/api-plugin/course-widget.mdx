# 課表桌面小工具

提供原生桌面小工具（App Widget），在手機主畫面或鎖定畫面顯示下一堂課的資訊。

## 資料流程

```
Dart (CourseData)
  │
  ▼ jsonEncode → MethodChannel
  │
  ├─ Android: SharedPreferences → AppWidgetManager 更新
  │
  └─ iOS:     App Group UserDefaults → WidgetCenter 更新
```

## Dart 端使用

### 初始化

iOS 需要設定 App Group ID，讓主 App 與 Widget Extension 共享資料。Android 會忽略此設定。

```dart
import 'package:ap_common_plugin/ap_common_plugin.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  if (!kIsWeb && (Platform.isIOS || Platform.isMacOS)) {
    await ApCommonPlugin.configure(
      appGroupId: 'group.com.yourapp.id',
    );
  }

  runApp(const MyApp());
}
```

### 更新小工具

在課表資料載入成功後呼叫：

```dart
final CourseData courseData = await fetchCourseData();

// 儲存快取
courseData.save(semester.cacheSaveTag);

// 推送至原生小工具
await ApCommonPlugin.updateCourseWidget(courseData);
```

### 清除小工具

在使用者登出時清除小工具資料：

```dart
await ApCommonPlugin.clearCourseWidget();
```

### 測試用假資料

在除錯模式下注入假課程資料，會建立兩筆課程（30 分鐘後、90 分鐘後）：

```dart
if (kDebugMode) {
  await ApCommonPlugin.setFakeCourseWidget();
}
```

:::tip 設定頁整合範例
可在設定頁面加入除錯區塊，方便測試：

```dart
if (kDebugMode) ...[
  SettingItem(
    text: '設定測試課表 Widget',
    subText: '注入 30 分鐘後的假課程資料',
    onTap: () async {
      await ApCommonPlugin.setFakeCourseWidget();
    },
  ),
  SettingItem(
    text: '清除課表 Widget',
    subText: '移除 Widget 中的課程資料',
    onTap: () async {
      await ApCommonPlugin.clearCourseWidget();
    },
  ),
]
```
:::

## Android 整合

Plugin 已包含完整的 `CourseAppWidgetProvider` 實作與 `AndroidManifest.xml` 註冊，**App 端不需要額外撰寫原生程式碼或修改 manifest**。

Plugin 內部的 manifest 已自動註冊以下 receiver，會在建置時合併至 App 的 manifest：

```xml title="ap_common_plugin/android/src/main/AndroidManifest.xml（Plugin 內部）"
<receiver
    android:name=".CourseAppWidgetProvider"
    android:exported="true">
    <intent-filter>
        <action android:name="android.appwidget.action.APPWIDGET_UPDATE" />
    </intent-filter>
    <meta-data
        android:name="android.appwidget.provider"
        android:resource="@xml/course_appwidget_info" />
</receiver>
```

### 原生實作細節

#### MethodChannel 處理（`ApCommonPlugin.kt`）

| Method | 行為 |
|--------|------|
| `getPlatformVersion` | 回傳 `"Android {版本號}"` |
| `configure` | **No-op**，App Group 僅 iOS 需要 |
| `updateCourseWidget` | 將 JSON 字串寫入 `SharedPreferences`（name: `ap_common_plugin`、key: `course_notify`），然後呼叫 `onUpdate()` 觸發所有已註冊的 widget 更新 |
| `clearCourseWidget` | 從 `SharedPreferences` 移除 `course_notify`，觸發 widget 更新 |

#### Widget Provider（`CourseAppWidgetProvider.kt`）

Widget 的顯示邏輯完全在原生端處理：

1. **讀取資料**：從 `SharedPreferences("ap_common_plugin")` 讀取 `course_notify` JSON 字串
2. **解析今日課程**：使用 `org.json.JSONObject` 解析，根據今天星期幾（ISO 8601：1=週一、7=週日）篩選課程
3. **找出下一堂課**：比對每堂課的 `startTime` 與目前時間，取時間差最小的正值
4. **顯示格式**：`"下一節課是{startTime}\n在 {location} 的 {title}"`（繁中），支援 `"HH:mm"` 和 `"HHmm"` 兩種時間格式
5. **特殊狀態**：
   - 無資料 → `"尚無課程資料"`
   - 今日無課 → `"太好了今天沒有任何課"`
   - 今日課程皆已結束 → `"太好了今天已經沒有任何課"`

#### Widget 佈局（`course_appwidget_provider_layout.xml`）

```
┌─────────────────────────┐
│  上課提醒          [🔄] │ ← 深藍背景 (#141e2f)，白色文字
├─────────────────────────┤
│                         │
│   下一節課是 09:00      │ ← 深色背景 (#121212)，灰色文字
│   在 EC5012 的 演算法   │
│                         │
└─────────────────────────┘
```

- **標題列**：「上課提醒」文字 + 右側重新整理按鈕
- **內容區**：點擊後開啟主 App
- **多語系**：支援英文（預設）與繁體中文（`values-zh-rTW`）

## iOS 整合

iOS 的 Widget Extension **無法**由 Flutter Plugin 自動打包（iOS 架構限制），需要在各 App 的 Xcode 專案中獨立建立。Plugin 提供範例程式碼作為參考。

### 原生實作細節

#### MethodChannel 處理（`SwiftApCommonPlugin.swift`）

| Method | 行為 |
|--------|------|
| `getPlatformVersion` | 回傳 `"iOS {版本號}"` |
| `configure` | 將 `appGroupId` 存入 `UserDefaults.standard`（key: `app_group_id`），供後續 method 讀取 |
| `updateCourseWidget` | 從 `UserDefaults.standard` 讀取 `appGroupId`，將 JSON 寫入 `UserDefaults(suiteName: appGroupId)`（key: `course_notify`），呼叫 `WidgetCenter.shared.reloadAllTimelines()` |
| `clearCourseWidget` | 從 App Group UserDefaults 移除 `course_notify`，重新載入所有 timeline |

:::info 資料儲存位置
- **App Group ID**：`UserDefaults.standard`，key `"app_group_id"`
- **課表 JSON**：`UserDefaults(suiteName: appGroupId)`，key `"course_notify"`

Widget Extension 透過相同的 App Group ID 存取 `UserDefaults`，讀取課表 JSON。
:::

#### Widget Extension（`CourseAppWidget.swift`）

Widget 使用 `TimelineProvider` 實作，顯示邏輯：

1. **讀取資料**：從 `UserDefaults(suiteName: appGroupId)` 讀取 `course_notify` JSON
2. **JSON 解碼**：使用 `JSONDecoder` 解碼為 `CourseData` struct
3. **計算星期**：`Calendar.current.dateComponents` 取得今天星期幾，轉換為 ISO 8601（1=週一、7=週日）
4. **找出下一堂課**：篩選今日 weekday 的課程，比對 `startTime` 與當前時間，取最小正值時間差
5. **Timeline 策略**：使用 `.atEnd` policy，timeline 結束後系統自動請求新的 timeline
6. **顯示格式**：
   - 有下一堂課 → `"下一節課是{startTime}\n在 {building}{room} 的 {title}"`
   - 今日無課 → `"太好了今天沒有任何課"`
   - 課程皆已結束 → `"太好了今天已經沒有任何課"`

#### Widget View 結構

| View | 用途 | 版本要求 |
|------|------|----------|
| `CourseAppWidgetEntryView` | 主畫面 widget，含標題列與課程內容 | iOS 14+ |
| `InlineWidgetView` | 鎖定畫面單行顯示（短文字） | iOS 16+ |
| `CourseTextWidgetEntryView` | 鎖定畫面矩形顯示 | iOS 16+ |
| `ViewSizeWidgetView` | 根據 `widgetFamily` 分派對應 View | — |

### 建立步驟

### Step 1：建立 Widget Extension

在 Xcode 中 **File → New → Target → Widget Extension**，命名為 `CourseAppWidget`。

### Step 2：設定 App Group

在主 App 和 Widget Extension 的 **Signing & Capabilities** 中加入相同的 App Group：

```
group.com.yourapp.id
```

:::caution 重要
此 ID 必須與 Dart 端 `ApCommonPlugin.configure(appGroupId:)` 傳入的值一致。
:::

### Step 3：建立資料模型

在 Widget Extension 中建立 `CourseData.swift`，欄位需與 Dart 端的 `CourseData` JSON 格式一致：

```swift title="CourseAppWidget/CourseData.swift"
import Foundation

struct CourseData: Codable {
    let courses: [Course]
    let timeCodes: [TimeCode]
}

struct Course: Codable {
    let code: String
    let title: String
    let className: String?
    let group: String?
    let units: String?
    let hours: String?
    let required: String?
    let at: String?
    let sectionTimes: [SectionTime]
    let location: Location?
    let instructors: [String]

    enum CodingKeys: String, CodingKey {
        case code, title, className, group, units, hours
        case required, at
        case sectionTimes, location, instructors
    }
}

struct Location: Codable {
    let building: String?
    let room: String?
}

struct SectionTime: Codable {
    let weekday: Int
    let index: Int
}

struct TimeCode: Codable {
    let title: String
    let startTime: String
    let endTime: String
}
```

:::caution JSON 欄位對應注意事項
- `required` 和 `at` 是 Swift 保留字/關鍵字，需透過 `CodingKeys` 明確對應
- `location` 必須為 `Optional`（`Location?`），因為部分課程可能沒有地點資訊；Dart 端 `Location` 的 `building` 和 `room` 雖為 required，但 JSON 層級可能為 null
- `sectionTimes` 對應 Dart 端屬性名為 `times`，透過 `@JsonKey(name: 'sectionTimes')` 映射為 JSON key `sectionTimes`
- `startTime` 格式支援 `"HH:mm"` 和 `"HHmm"` 兩種
:::

### Step 4：實作 Timeline Provider

從 App Group UserDefaults 讀取課程資料，計算今天的下一堂課：

```swift title="CourseAppWidget/CourseAppWidget.swift"
struct Provider: TimelineProvider {
    func getTimeline(
        in context: Context,
        completion: @escaping (Timeline<SimpleEntry>) -> Void
    ) {
        let defaults = UserDefaults(
            suiteName: "group.com.yourapp.id"
        )
        let json = defaults?.string(forKey: "course_notify")

        // 解碼 JSON
        guard let data = json?.data(using: .utf8),
              let courseData = try? JSONDecoder().decode(
                  CourseData.self, from: data
              )
        else {
            // 無資料時的預設顯示
            let entry = SimpleEntry(text: "尚無課程資料")
            let timeline = Timeline(
                entries: [entry], policy: .atEnd
            )
            completion(timeline)
            return
        }

        // 計算今天星期幾 (ISO 8601: 1=週一, 7=週日)
        let components = Calendar.current.dateComponents(
            in: TimeZone.current, from: Date()
        )
        let weekday = components.weekday == 1
            ? 7
            : (components.weekday ?? 1) - 1

        // 找出下一堂課...
    }
}
```

## macOS 整合（規劃中）

macOS 從 Big Sur（11.0）起支援 WidgetKit，與 iOS 使用相同的 SwiftUI + WidgetKit 技術棧。現有的 iOS Widget 程式碼可大幅複用。

### 與 iOS 的差異

| 項目 | iOS | macOS |
|------|-----|-------|
| Widget 顯示位置 | 主畫面、鎖定畫面 | 通知中心、桌面（Sonoma+） |
| 最低版本 | 14.0 | 11.0 |
| UIColor | ✅ 可用 | ❌ 需改用 `Color` 或 `NSColor` |
| App Group 設定 | Signing & Capabilities | 相同 |

### 移植要點

1. `CourseAppWidget.swift` 中的 `UIColor` 需替換為跨平台寫法：
   ```swift
   // 原本（僅 iOS）
   Color(UIColor(red: 0.15, green: 0.45, blue: 1.00, alpha: 1))

   // 跨平台
   Color(red: 0.15, green: 0.45, blue: 1.00)
   ```
2. `SwiftApCommonPlugin.swift` 的 MethodChannel 邏輯在 macOS 上相同
3. Dart 端 `configure()` 已支援 macOS（`Platform.isMacOS` 判斷）

### 不支援的平台

| 平台 | 原因 |
|------|------|
| **Windows** | Widget Provider 需 C++/WinRT 實作 `IWidgetProvider` COM 介面，Flutter 生態系無可用方案 |
| **Linux** | 桌面環境碎片化（KDE 用 QML、GNOME 用 GJS），無統一 widget 框架 |
| **Web** | PWA 不支援原生 widget，W3C 提案仍在早期階段 |
